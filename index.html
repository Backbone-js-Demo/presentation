Backbone.js
  • Eher Bibliothek, kein Framework
  • Baukasten für Single-Page-Apps
  • Verwendet Model View Controller (MVC)
  • Bottom-Up-Prinzip
  • kümmert sich nur um Kernfunktionalitäten
  • Plugin-System
    • Start eines Projekts ist Architektur frei wählbar
    • verschiedene Frameworks als Backbone.js-Aufsatz (Backbone.Marionette)

  • Voraussetzung
    • Ajax-Bibliothek (z.B. jQuery)

Ursprung von Backbone.js
  • Nachfolger von Flash
    • unzureichende Performance
    • auf iPhone und iPad nicht unterstützt
    • Ist proprietäre Technologie
Das MVC-Framework

Model: enthält Daten und Programmlogik, diese sind nicht nach außen sichtbar.

View: gibt die Ansicht des Models aus, die benötigten Daten kommen vom Model

Controller: steuert den Verarbeitungsprozess. Nimmt Userinput entgegen und ändert das Model ab.

Single Page Applications
  • Besteht aus einzelnem HTML-Dokument
  • Inhalt wird dynamisch nachgeladen
  • Datenaustausch mittels JSON oder XML
  • Beispiel Webseiten
    • Facebook
    • Soundcloud
  • Techniken
    • Ajax
    • Flash
    • WebSockets
Eigenschaften von Backbone.js
  • Strukturiert den Javascript-Code durch Einteilung in MVC-Framework
  • Ermöglicht Single Page Applications in Javascript umzusetzen
  • Bietet die Möglichkeit Teile des UI zu verändern ohne die gesamte Seite neu zu laden
  • unterstützt REST APIs (Read, Write, Delete)
Typischer Aufbau 

				<!doctype html>
				<head>
				  <title>Backbone.js Example</title>
				</head>
				<body>
				    <script src="json2.js"></script>
				    <script src="jquery-1.11.1.js"></script> 
				    <script src="underscore.js"></script>
				    <script src="backbone.js"></script>
				    <script src="backbone.localStorage.js"></script>
				    <script src="addressbook.js"></script>
					</body>
				</html>
