Backbone.js
Eigenschaften

  • Eher Bibliothek, kein Framework
  • strukturiert Code durch Einteilung in MVC-Framework
  • ist ein Baukasten für Single-Page-Apps
  • Bottom-Up-Prinzip
  • kümmert sich nur um Kernfunktionalitäten, Plugin-System
  • Bietet die Möglichkeit Teile des UI zu verändern ohne die gesamte Seite neu zu laden
  • unterstützt REST APIs (Read, Write, Delete)
Backbone.js
Kritik

  • minimalistischen Herangehensweise, daher kaum Zusatzfunktionen
  • häufige Funktionen müssen selbst programmiert werden

Voraussetzungen

  • Utility-Bibliothek underscore.js unbedingt notwendig
  • REST-Funktionen benötigen eigene Frameworks (z.B. jQuery, json2)
Ursprung von Backbone.js

im Jahre 2010 von Jeremy Ashkenas für DocumentCloud, eine Dateiverwaltungs-Software, entwickelt


wurde als Nachfolger von Flash entwickelt


Kritik an Flash, die Backbone.js beheben soll:

  • unzureichende Performance
  • auf iPhone und iPad nicht unterstützt
  • ist proprietäre Technologie
Das MVC-Framework

Model: enthält Daten und Programmlogik, diese sind nicht nach außen sichtbar.

View: gibt die Ansicht des Models aus, die benötigten Daten kommen vom Model

Controller: steuert den Verarbeitungsprozess. Nimmt Userinput entgegen und ändert das Model ab.


nicht alle Teile müssen bei Backbone.js vorhanden sein

Single Page Applications
  • Besteht aus einzelnem HTML-Dokument, Inhalt wird dynamisch nachgeladen
  • Datenaustausch mittels JSON oder XML

Beispiele
  • Facebook
  • Soundcloud

Techniken
  • Ajax
  • Flash
  • WebSockets
